# Data files

This folder holds YAML data files that are imported directly into Astro pages at build time.
No special Jekyll `_data` folder or plugin needed — just import the file in your `.astro` page:

```ts
import citations from '@/data/citations.yaml';
import resources from '@/data/resources.yaml';
```

## Files

| File | Source | Description |
|---|---|---|
| `citations.yaml` | Generated by Manubot (`_cite/cite.py`) | Full citation objects for Publications page |
| `sources.yaml` | Maintained manually | Input DOIs/ORCIDs for Manubot |
| `orcid.yaml` | Maintained manually | ORCID → member mapping |
| `projects.yaml` | Maintained manually | Research project cards |
| `resources.yaml` | Maintained manually | Resources page links |
| `events.yaml` | Maintained manually | Upcoming events |
| `types.yaml` | Maintained manually | Publication type labels/filters |

> **Note:** `citations.yaml` and `sources.yaml` are still written/read by the
> `_cite/` Manubot pipeline unchanged. The only difference is that Astro reads
> the output at build time instead of Jekyll.
